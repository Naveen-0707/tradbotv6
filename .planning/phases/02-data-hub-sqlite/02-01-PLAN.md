# Step Plan: 02-01: DB Schema and setup

## Task
1. Install `better-sqlite3`.
2. Create `db.js`.
3. Set up the schema for `trades` and `signals`.

## Details
- Use `db.exec` for table creation.
- Table `trades`: id (TEXT primary), obj (TEXT JSON), date (TEXT), name (TEXT), status (TEXT). We can map the whole object internally or use normalized columns. The easiest mapping is to stringify the whole trade object into `data` column, with indexed columns for easy queries: `id`, `date`, `name`, `status`, `paper`.
- Table `signals`: `id`, `date`, `name`, `data` (JSON).

## Verification
- Test running `node db.js` to initialize an empty database and verify file creation.
